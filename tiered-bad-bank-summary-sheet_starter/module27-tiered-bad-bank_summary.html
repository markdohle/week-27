Local State

"State" refers to changes in your program based on user input or interaction. An example of state changing would be when a user submits a login form that clears the form fields upon submission.

Converting Components to React

Refactoring your plain JavaScript code so that a frontend framework like React can abstract it can to save you time. This is why you had to do it in the Bad Bank exercise. Here are the necessary steps:

  • Routing
  • State management
  • Data binding

Routing in React

In the past, you may have routed your applications using 'onClick' events, moving from page to page when a user clicked a particular link. However, the browser doesn't automatically remember what page the user has come from, so simple functionality like navigating with a back button is not possible. Now, you can use HashRouter and React to create routes and manage browser states. For more information, refer to the HashRouter documentation.

Node Server

A Node server allows you to serve files through an external router such as Express using the HTTP protocol. It allows interaction between your files and those of the users. When users interact with your website on the front end, it makes sure that your database is updated. Also, it ensures that data is retrieved from the database and displayed to the users.

Database Types

There are several types of databases to choose from. This module, we will examine relational databases and document-style databases. Relational databases store an object's data in separate tables, while document-style databases store all the data for a given object in one database instance, using a format similar to a JavaScript object.

MongoDB

MongoDB is the document-style database that was used in the Bad Bank exercise. Like other document databases, MongoDB uses JSON-like documents called "BSON" (binary representation of JSON objects) with optional database schemas.
